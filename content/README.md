# 别调Prompt了，去写Loop

> Loop Engineering：从提示词工程到循环工程的范式跃迁。

## 关于本专栏

2026 年 6 月，Google Chrome 团队的 Addy Osmani 正式命名了一个概念--**Loop Engineering（循环工程）**。几乎同一时间，黄仁勋说"现在根本没有人写 Prompt 了，新时代的核心工作是编写和管理 loop"。

这不是凭空出现的新词。从 Copilot 的代码补全，到 Cursor 的聊天式编程，到 Claude Code 的代理模式，再到今天的循环工程，软件工程正在经历一场范式的根本跃迁：从"人 prompt 代理"走向"人设计 loop，loop prompt 代理"。

本书分中英双语版本，完整梳理 Loop Engineering 的来龙去脉：

- **第一部分 范式革命**：为什么是现在，Loop 工程的三个先决条件
- **第二部分 核心架构**：循环的骨架、状态机、反馈回路、错误恢复
- **第三部分 设计与实践**：从需求到 PR 的完整循环设计、真实工程案例
- **第四部分 框架与生态**：主流 Loop 框架对比、MCP 协议、工具链选型
- **第五部分 未来与组织**：当 Loop 成为标配，工程师的新角色
- **附录**：术语表、检查清单、推荐阅读

完整目录见 [INDEX.md](INDEX.md)。
